发表于2023-01-06|Godot学习日志
# 🔵1. 认识编辑器布局 # 🔷左侧边栏 🔹文件系统（FileSystem）：显示当前项目中所有资源文件 🔹导入（Import）：显示文件系统分栏中所选资源文件的导入设置 🔹场景（Scene）：按照层级结构显示当前场景中所有节点，可以简单理解为一个游戏里的场景 / 关卡 # 🔷右侧分栏 🔹属性（Inspector）：显示场景分栏中所选节点的属性 🔹节点（Node）：显示与场景分栏中所选节点对应的一些信息 # 🔷中间面板 2D：编辑 2D 场景的窗口 3D：相对的用于编辑 3D 场景 脚本（Script）：Godot 内置的脚本编辑器，可以使用外置 IDE 替代 资源库（AsseLib）：可以浏览搜索网络上的资源 # 🔷底部面板 调试控制台 动画编辑器 混音器 # 🔵2. 认识游戏对象 🔷游戏对象（Node）：即是场景里包含的一切内容，Godot 的一切操作都是基于 Node 实现 🔷所有节点都拥有以下特性 可变更的名称 可编辑的属性 每帧都可以接受回调进行更新 使用新的属性 / 函数进行扩展 可添加为其他节点的子节点 节 ...
